# FlowDoChat タスクライフサイクル仕様書

> **関連文書:** `憲法.txt`（罪悪感ゼロ）, `アプリの芯.txt`（パートナー感）

---

## 📌 基本思想

> **「ToDoが溜まらない」= 罪悪感ゼロ**
> 
> このアプリのタスクは「管理するもの」ではなく「今に集中するためのガイド」である。
> ユーザーが過去の未完了に追われることは絶対にない。

---

## 🔄 タスクの一生

```
┌─────────────────────────────────────────────────────────────┐
│  1. 生成 → 2. 保存 → 3. 消化 → 4. 記録 → 5. リセット        │
└─────────────────────────────────────────────────────────────┘
```

| Step | 名称 | 動作 |
|---|---|---|
| 1 | **生成** | AIとの会話 or 手動追加 → チャット内にカード表示 |
| 2 | **保存** | TODAY'S FLOW に追加、日付は「今日」のみ |
| 3 | **消化** | チェックで完了、Haptic + AIが褒める |
| 4 | **記録** | ヒートマップに永久保存 |
| 5 | **リセット** | 日付変更で未完了は自動削除（Overdue概念なし） |

---

## 🚫 存在しない概念

| 概念 | 理由 |
|---|---|
| Overdue（期限切れ） | 罪悪感ゼロのため |
| 繰り越し | ToDoが溜まる原因のため |
| 優先度（高/中/低） | 複雑化防止のため |

## 🌅 Day 2 朝フロー

### AIの第一声パターン

| シナリオ | AIの挨拶 |
|---|---|
| 昨日頑張った | 「昨日は2つ片付けたね、お疲れさま！」 |
| 未完了あり | 「昨日の○○、今日も気になってる？」 |
| あまり使わなかった | 「昨日はゆっくりしてた？」 |

### ユーザー返答への対応

| 返答 | AIの対応 |
|---|---|
| 「昨日の続きやる」 | タスク復活 |
| 「別のことやる」 | 新規タスク生成 |
| 「今日はダルい」 | 軽めを提案 or 休息勧める |

---

## 🧠 AIが覚えていること

| 情報 | 保持期間 | 用途 |
|---|---|---|
| 昨日のタスク（完了） | 永続（ヒートマップ） | 褒める材料 |
| 昨日のタスク（未完了） | 翌日朝まで | 提案材料 |
| パートナー設定 | 永続 | 返答スタイル |

---

## 📊 Day 2 成功の定義

1. **開く理由がある** → AIが何て言ってるか見たい
2. **罪悪感がない** → 昨日の未完了を見て「うわ...」とならない
3. **すぐ会話が始まる** → 沈黙の画面にならない
4. **5分以内にタスクが生成される** → 今日やることが明確になる
